# Codex Audit — Cycle 1
# 2026-03-29T02:34:29.540348+00:00

1. P0: The “atomic” write paths are not atomic. [`src/db.py`](src/db.py#L197) and [`src/db.py`](src/db.py#L225) call `conn.commit()` internally, so the outer transactions in [`src/runner.py`](src/runner.py#L374) and [`src/analysis.py`](src/analysis.py#L411) cannot roll back a partial run/reparse. A failure after `insert_run()` or after some `insert_tool_call()`s leaves silent partial state: orphaned `runs` rows, partially replaced `tool_calls`, and misaligned aggregates.

2. P1: Compound Bash commands are misclassified as read-only when the first token is read-only. In [`src/trace_collector.py`](src/trace_collector.py#L174), `|` gets special handling only if there is no `&&` or `;`. Then [`src/trace_collector.py`](src/trace_collector.py#L196) returns read-only based on the first token alone. Cases like `grep foo x && rm y` or `git diff; git apply p.diff` will be recorded as non-state-modifying and dropped from analysis.

3. P1: `xargs` is treated as read-only in the base classifier. [`src/trace_collector.py`](src/trace_collector.py#L61) includes `xargs` in `READ_ONLY_COMMANDS`, and the single-command path at [`src/trace_collector.py`](src/trace_collector.py#L196) therefore classifies `xargs rm`, `xargs touch`, etc. as read-only. That undercounts state-modifying calls and weakens gating exactly on shell fan-out mutations.

4. P1: Both validation scripts overwrite the primary `runs.task_success` label in place. [`src/docker_validate.py`](src/docker_validate.py#L127) and [`src/swebench_validate.py`](src/swebench_validate.py#L23) / [`src/swebench_validate.py`](src/swebench_validate.py#L140) mutate the same field the runner originally set. This silently destroys provenance between “workspace validation at run time” and “post hoc Docker/SWE-bench validation,” so downstream analysis cannot distinguish the original outcome from later re-evaluations.

5. P1: Patch reconstruction for SWE-bench can silently produce the wrong diff. [`src/swebench_validate.py`](src/swebench_validate.py#L105) replays `Edit` operations with `str.replace(old, new, 1)` and simply skips an edit if `old_string` is no longer present. Repeated substrings, shifted context from earlier edits, or mismatched edit targets yield corrupted or incomplete patches with no error, which can flip validation results and contaminate `task_success`.

6. P2: Feature extraction commits one row at a time. [`src/db.py`](src/db.py#L258) commits inside `update_features()`, and [`src/analysis.py`](src/analysis.py#L157) calls it per tool call. An interrupted analysis run leaves mixed old/new feature values within the same dataset, which is a silent analysis-integrity problem for correlations and summaries.
