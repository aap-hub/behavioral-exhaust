# 1. Introduction

Autonomous coding agents now routinely solve software engineering tasks that, until recently, required human developers. On benchmarks like SWE-bench Lite (Jimenez et al., 2024), frontier models resolve 40--50% of real-world bug reports with no human intervention. Yet deploying these agents outside controlled benchmarks demands something benchmarks do not measure: knowing *when not to trust the agent's output*. A human developer who watches an agent edit the wrong file, chase a red herring for twenty minutes, or silently break an unrelated test needs a signal --- ideally before the damage is done --- that the current trajectory is going wrong.

The difficulty is that the standard tools for estimating model reliability are largely unavailable in agentic settings. Log-probabilities, which underpin calibration in classification tasks, are not exposed by most agent-serving APIs. Sampling-based approaches like self-consistency require multiple completions of long, expensive trajectories. And the multi-turn structure of agent interaction --- where each tool call conditions on the results of previous calls --- means that a confidence estimate at step $t$ must somehow account for the accumulated state of steps 1 through $t-1$, a problem that single-turn calibration methods were not designed to address.

Three families of approaches have emerged. The first elicits confidence scores directly from the model, either through verbalized probability estimates or softmax-derived scores. This is unreliable for agents: self-reported confidence is poorly calibrated when the model's context accumulates errors over a multi-turn trajectory (Kadavath et al., 2022; Xiong et al., 2024; Xuan et al., 2026). The second deploys external critics --- a second model, a formal verifier, or a human reviewer --- to evaluate the agent's actions. This works but scales poorly: every gated decision requires an additional inference call, adding latency and cost proportional to the agent's action count. Cuadron et al. (2025) demonstrate the potential of this approach with Shepherd, using LLM-as-judge to select among candidate trajectories, but the cost of running a discriminator on each candidate limits practical deployment. The third approach, and the least explored, asks whether the behavioral exhaust that agents already produce --- reasoning text, tool-call sequences, error patterns --- contains extractable signal about task success. This signal is zero-cost (the exhaust already exists in the trace), non-invasive (no prompt modification required), and does not add latency to the agent's execution.

This paper investigates the third approach. We extract 24 behavioral features from agent reasoning traces across 373 validated runs (183 Claude Sonnet 4.6, 190 Codex GPT-5.4) on 75 SWE-bench Lite tasks, using only regex and string matching --- no additional model calls. The features span three tiers: structural properties of the tool-call sequence (error streaks, edit timing), linguistic properties of reasoning text (deliberation length, contrastive markers), and domain-specific properties that bridge reasoning and action (diagnostic precision, file-reference density in thinking blocks). All runs are independently validated by replaying agent edits on fresh environments and running the target test suite; agent self-report is never used for labeling.

Our central finding is that **behavioral structure, not linguistic hedging, is associated with task success**. Academic hedging vocabulary --- the "perhaps," "might," "possibly" words that predict reasoning errors in question-answering (Vanhoyweghen et al., 2025) --- carries zero signal in coding agent traces ($\rho \approx 0$, $p = 0.60$). What does carry signal is how the agent *structures* its reasoning: how long it deliberates per action, how early it encounters errors, and whether its internal thinking contains precise references to the code artifacts it will modify.

These associations are model-specific in ways that have practical consequences. Across the two models in our study, only structural features (consecutive error streaks, early error rate) transfer. Every linguistic feature that survives rigorous testing is specific to one model, and one feature --- contrastive marker density --- shows a confirmed polarity flip: it is positively associated with success in Sonnet ($\rho = +0.282$, $p = 0.008$) but negatively associated with success in Codex ($\rho = -0.264$, $p = 0.020$), within the same repository and task set. A naive monitor that treats contrastive language as a positive signal would be correct for one model and wrong for the other.

The signal also degrades over the trajectory. In the early and middle thirds, behavioral features discriminate pass from fail. In the final third, they do not ($\rho \approx 0$, $p > 0.6$). By the time an agent is deep into a trajectory, it has either found the fix or committed to a wrong approach; its reasoning structure no longer discriminates. This has a direct implication for gating: behavioral gates should focus on early and mid-trajectory decisions, where the signal is strongest.

We report five contributions:

1. **A behavioral feature extraction pipeline for agent reasoning traces.** Twenty-four features across three tiers, computed from existing trace data with zero API overhead. All features use per-call mean aggregation to control for the task-length confound (failing tasks produce 38% more tool calls than passing tasks).

2. **A cross-model comparison revealing model-specific feature associations and one confirmed polarity flip.** Despite broadly similar aggregate performance (Sonnet 49%, Codex 41% on matched tasks) and high task-level agreement (87%, Cohen's $\kappa = 0.74$), the models' behavioral signatures diverge sharply. Contrastive language ("instead," "rather") is associated with diagnostic narrowing in Sonnet and approach abandonment in Codex. Only two features --- both structural (error streaks, early error rate) --- show consistent associations across models. This implies that any behavioral gating system must be calibrated per model; there is no universal uncertainty lexicon for coding agents.

3. **A thinking/message double dissociation.** The word "actually" in internal thinking blocks correlates negatively with success (genuine confusion), while the same word in agent messages correlates positively (performative self-correction). More broadly, internal doubt lives in thinking blocks and expressed correction lives in messages. This extends work on chain-of-thought faithfulness (Arcuschin et al., 2025) by showing that the two layers carry structurally different and sometimes opposite information.

4. **A domain-specific negative result on Hyland hedging.** The 209-term academic hedging vocabulary (Hyland, 2005) that is predictive in QA reasoning chains is completely non-predictive for coding agents (rho near zero, p = 0.60). Coding agents express uncertainty through structural behavior, not through hedging words. This contrasts directly with findings in question-answering (Vanhoyweghen et al., 2025) and establishes that uncertainty signal extraction is domain-dependent.

5. **A CWC methodology for controlling repository-level heterogeneity.** Repository pass rates in our sample range from 0% (scikit-learn) to 84% (django). Naive pooled analysis inflates effect sizes. We combine within-repository Spearman correlations via Fisher's method, then validate with within-repository permutation tests (10,000 permutations). This methodology exposed our own headline finding as confounded: reasoning-to-action alignment, originally our strongest result, does not survive permutation (p = 0.35) and is retracted. We report this retraction as evidence of the methodology's value, not as a weakness. The corrected headline --- deliberation length as the most robust Sonnet-specific feature (permutation p = 0.0004) --- is more conservative and more credible.

Two scope boundaries deserve emphasis. First, all findings are associational. We report rank correlations between behavioral features and task outcomes, controlled for repository heterogeneity and validated by permutation. We have not evaluated held-out accuracy, calibration, or any out-of-sample metric. The contribution of this paper is establishing that extractable signal *exists* in behavioral exhaust, not that it can yet be used for reliable prediction. Second, we study two models on one benchmark in one domain. The features, polarities, and interactions we observe may not generalize to other models, other benchmarks, or other task types. We treat model-specificity as a finding, not a limitation: it tells us something fundamental about the nature of behavioral signal in agent reasoning, namely that it reflects model-specific reasoning strategies rather than universal properties of task difficulty.
