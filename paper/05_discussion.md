# 5. Discussion

## 5.1 What the Human Doesn't See

The strongest signals associated with agent task success live in layers that are invisible to a human reviewing the agent's output. Three categories of signal are systematically hidden from view.

First, the most diagnostic behavioral features reside in internal reasoning, not in agent messages. Sonnet's extended thinking blocks contain roughly 10x more text per run than its polished messages (~11,000 vs ~1,200 characters). Within those thinking blocks, tentative language density shows a significant negative association with success (rho = -0.346, p = 0.003), while the same feature extracted from messages does not reach significance (rho = -0.204, p = 0.083). A human reading only the agent's messages -- which is the default experience for users of chat-based coding assistants -- would miss the strongest individual linguistic signal in our dataset. The diagnostic precision feature, which measures the density of code-specific references (function names, file paths, line numbers) per thousand characters of internal reasoning, distinguishes Sonnet-pass from Codex-fail tasks at 4-16x ratios. None of this is visible in the message stream.

Second, the strongest practical association we observe is not a single feature but an interaction between two features. In Sonnet runs on sympy, the combination of high deliberation length and high diagnostic precision yields a 73% pass rate, while either feature alone yields 23-32%. The interaction bonus of +0.56 far exceeds any main effect. A monitoring system that tracks individual features in isolation would miss this nonlinear structure entirely. An agent that reasons at length but imprecisely (23% pass) actually performs worse than one that barely reasons at all (38% pass). Length without precision is a marker of unproductive rumination, not careful analysis. This is precisely the kind of finding that contradicts the naive intuition that "more thinking is better."

Third, the polarity of linguistic features is model-specific in ways that would mislead a human observer applying intuition from one model to another. Contrastive language density (words like "however," "but," "instead") is significantly associated with success in Sonnet (rho = +0.282, p = 0.008) and with failure in Codex (rho = -0.264, p = 0.020) within the same repository. A human who learned to read Sonnet's contrastive language as a positive sign would misinterpret the identical pattern in Codex output.

Together, these three categories -- hidden layers, hidden interactions, hidden polarities -- define the scope of what behavioral exhaust mining can add beyond human review. The information exists in the traces that agents already produce. The challenge is extraction and interpretation, not collection.

## 5.2 Implications for Gating

The Phase 0 findings, if they survive intervention testing in Phase 1, have four implications for the design of behavioral gates that modulate agent autonomy.

**Gates must be model-specific.** The confirmed polarity flip on contrastive marker density, combined with the entirely different feature sets that survive CWC and permutation testing for each model (Sonnet: deliberation_length, first_edit_position; Codex: think_t_compat_fraction, instead_contrast_density), rules out universal feature thresholds. A gate calibrated on Sonnet's behavioral exhaust would misclassify Codex runs on at least one feature, and vice versa. This model-specificity also implies that any deployed gate would require recalibration when the underlying model is updated -- a practical constraint that limits the utility of behavioral gating in production systems where model versions change frequently.

**Gates should focus on early trajectory.** Signal quality degrades over the course of a run. In both deliberation length and the retracted alignment feature, within-repo correlations with success are significant in the early (0-33%) and middle (33-67%) trajectory segments but collapse to near-zero in the late segment (67-100%). Early error rate, one of only two cross-model features, is by definition an early-trajectory measurement. The implication is that a behavioral gate has the most discriminative power when it intervenes early. By the late trajectory, the agent has either converged on a solution or committed to a failing approach, and reasoning features no longer differentiate the two. This temporal structure is consistent with the general finding that failing runs produce 38% more tool calls than passing runs: the late trajectory of a failing run is dominated by increasingly unproductive iterations that look qualitatively similar regardless of the eventual outcome.

**Nonlinear gates should outperform linear ones.** The deliberation-by-precision interaction in Sonnet suggests that a gate checking whether both conditions are met simultaneously would substantially outperform a gate checking either condition in isolation. In the quadrant analysis, the high-both cell achieves 73% pass, while the best single-feature cell achieves only 32%. A linear combination of the two features cannot capture this interaction without an explicit interaction term. Whether this advantage survives out-of-sample evaluation is unknown -- the quadrant analysis operates on approximately 22 runs per cell and has not been tested with a formal interaction model. But the magnitude of the interaction bonus (+0.56) is large enough to motivate the more complex gate design.

**Gates must specify which layer to extract from.** The thinking/message dissociation means that a feature extraction pipeline must explicitly choose whether to read internal reasoning or agent messages. Conflating the two layers would cancel real signals: "actually" in thinking correlates negatively with success (r = -0.253), while "actually" in messages correlates positively (r = +0.494). A pipeline that pools all text would observe a diluted or null effect. This is a practical implementation requirement, not merely a theoretical concern -- the parser bug that initially caused our pipeline to read message text instead of thinking blocks masked an entire layer of signal until it was corrected.

## 5.3 The Polarity Flip

The confirmed polarity flip on contrastive marker density deserves closer examination because it illustrates why model-specific calibration is not merely advisable but mandatory.

In Sonnet's sympy runs, words like "however," "but," and "instead" appear within diagnostic reasoning that narrows the gap between observed and expected behavior. A typical Sonnet pass pattern reads: "The function returns X, but the expected behavior is Y." The contrastive marker signals convergence -- the agent is building toward a diagnosis by contrasting what it sees with what should be. In Codex's sympy runs, the same words appear within strategic pivots: "However, let me try a different approach instead." Here the contrastive marker signals divergence -- the agent is abandoning one strategy for another without resolving the previous attempt.

The same surface form serves different cognitive functions in different architectures, producing opposite correlations with success. This finding has a methodological implication: any study that pools linguistic features across models without checking for polarity consistency will wash out real signals. In our combined-model analysis, contrastive density has no significant association with success precisely because the positive Sonnet signal and the negative Codex signal cancel. The two structural features that do survive cross-model analysis (fail_streak_max, early_error_rate) survive specifically because they have consistent polarity in both models.

There is, however, a confound that must be acknowledged. Sonnet and Codex produce internal reasoning in architecturally different formats. Sonnet's extended thinking blocks cover 100% of runs but only 28.8% of individual tool calls, while Codex's reasoning items cover 100% of runs and 60.1% of calls. The per-call coverage difference means that the linguistic features extracted from per-call reasoning text are computed over different samples of the agent's behavior in each model. It is possible that the polarity flip partly reflects this sampling difference rather than a genuine difference in how the models use contrastive language. Phase 1 should examine whether the flip persists when features are extracted only from the thinking-block layer, where coverage is 100% for both models.

Two additional features -- backtrack_count and metacognitive_density -- show suggestive directional reversals (opposite signs across models, but only one side reaches significance). These are hypotheses for Phase 1 testing, not confirmed findings. If they replicate, they would strengthen the case that model-specific calibration is a fundamental requirement for behavioral exhaust mining.

## 5.4 Relationship to the Faithfulness Literature

The thinking/message dissociation we observe is consistent with Chen et al.'s (2025) finding that chain-of-thought content is unfaithful approximately 75% of the time. Our data adds a nuance to this finding: even when the content of internal reasoning diverges from what the agent expresses publicly, the *structure* of both layers still carries signal associated with task outcomes. Unfaithfulness, in this framing, is not the absence of signal but the presence of *different* signal in different layers.

The "actually" dissociation is the clearest illustration. In internal reasoning, "actually" marks genuine pre-action confusion -- a point where the agent reconsiders its approach, which is associated with failure (r = -0.253). In agent messages, "actually" marks performative self-correction -- a display of metacognitive competence for the user, which is associated with success (r = +0.494). The same word is diagnostic in both layers, but the diagnosis points in opposite directions. A faithfulness-focused analysis that asked "does the thinking match the message?" would correctly flag the mismatch but would miss the fact that both layers are independently useful.

This suggests a reframing of the faithfulness question for agent monitoring: rather than asking whether the agent's public reasoning faithfully represents its private reasoning (it does not), we should ask what each layer reveals about the agent's state. Internal reasoning reveals genuine uncertainty; expressed reasoning reveals the agent's model of what successful problem-solving looks like. Both are useful for reliability estimation, provided they are extracted and interpreted separately.

Chen et al. also observed that unfaithful chains of thought are systematically longer than faithful ones. Our finding that deliberation_length is positively associated with success in Sonnet may appear to contradict this, but the two results measure different things. Chen et al. measure total CoT length in a question-answering setting where longer reasoning often reflects unnecessary elaboration. We measure per-call reasoning length in an agent setting where longer reasoning per action reflects more thorough consideration before each step. The per-call normalization is critical: total reasoning length correlates with total tool calls (which correlates with failure), but per-call reasoning length correlates with success.

## 5.5 Implications for Agent Safety Research

The central practical finding of this study is that behavioral exhaust -- the reasoning text and tool-call sequences that agents already produce -- carries signal associated with task outcomes, and that this signal can be extracted at zero additional API cost. No prompt modification is required. No additional model calls are needed. The features are computed from the trace data that any instrumented agent already generates.

For Anthropic's agent safety research, this has several implications. First, behavioral features could serve as runtime monitoring signals. The structural features (error streaks, early error rate) are model-agnostic and could be deployed as lightweight anomaly detectors without model-specific calibration. When an agent accumulates three or more consecutive errors, or when its early-trajectory error rate exceeds a threshold, the system could flag the run for human review or trigger an intervention. These features survived all our statistical tests and showed consistent direction across both models.

Second, the model-specificity of linguistic features means that any behavioral monitoring system calibrated on one model version will require recalibration when the model is updated. This is a maintenance cost that production systems must budget for. The magnitude of recalibration is not trivial: a feature that is the strongest Sonnet-specific predictor (deliberation_length) is not significant in Codex, and a feature that flips polarity entirely (contrastive density) would produce actively wrong classifications if the calibration from one model were applied to another. The only features that appear to transfer are structural ones that do not depend on the content of reasoning text.

Third, our findings are associational, not predictive. We have not demonstrated that a gate built on these features would improve outcomes, nor have we computed the operating characteristics (sensitivity, specificity, false positive rate) that would be required for deployment. The distance between "this feature correlates with success" and "this feature can be used to improve agent reliability" is substantial and must be bridged by the intervention testing planned for Phase 1. Anthropic's own methodology for model evaluations (2025) emphasizes paired comparison designs and clustered standard errors for exactly this kind of within-subject testing. Phase 1 should adopt that framework.
