# UGA vNext Protocol (Codex-Proposed, Stripped-Down)

Generated by Codex (gpt-5.3-codex) on 2026-03-28 as an adversarial redesign.

## Research Question
Do pre-tool-call behavioral signals from coding-agent reasoning traces predict next-call correctness, and does using that prediction as a real-time gate improve objective task outcomes?

## Arms (2 only)
- **Arm A (Ungated):** standard agent loop, no intervention
- **Arm B (Gated):** identical loop + risk scorer p(incorrect_next_call) from trace features; if p > τ, enforce deterministic guard (non-mutating diagnostic → recompute → block if still high)

## Task Suite (96 tasks, frozen, objective oracles)
- 48 code-fix (hidden tests)
- 24 refactor/safety (AST/lint/static checks)
- 24 tool-orchestration (file + shell + patch milestones)
- Paired design: each task in both arms with same seed
- Pilot: 20 tasks for calibration. Holdout: 76 for confirmatory.

## Endpoints (objective, no subjective labeling)
- **Primary: Task Success Rate (TSR)** — binary pass/fail from executable oracle
- **Secondary: Incorrect Tool-Call Rate (ITCR)** — machine-labeled (runtime failure, score decrease, policy violation)
- **Efficiency:** wall-clock, tokens, tool calls per solved task
- **Predictive quality:** AUROC/AUPRC of risk scorer
- **Overhead:** median runtime increase

## Features (deterministic, no annotation)
Step index (normalized), prior failure streak, tool-switch rate, retry count, argument edit distance, plan-revision count, uncertainty markers count, elapsed think-time.

## Analysis Plan
- Paired TSR comparison, stratified by task type, effect size + 95% CI
- Late-bin control: normalized step-index fixed effects + quartile reporting + truncation sensitivity
- Clustered SE / block bootstrap by episode
- Holm-Bonferroni for confirmatory secondaries
- One primary endpoint, two confirmatory secondaries

## Positive Result Criteria (pre-registered)
- ΔTSR ≥ +8pp, 95% CI excluding 0
- ITCR reduced ≥ 20% relative
- Overhead ≤ 15%
- AUROC ≥ 0.70 overall, ≥ 0.65 in 3/4 position quartiles

## Timeline (4 weeks)

- Week 2: implement adapters, logging, scorer, gated policy
- Week 3: pilot calibration (20 tasks), lock threshold
- Week 4: full run (holdout), analysis, report + demo video
